### 2.2 규정 파일 형식 (프론트매터 포함)

각 규정 파일은 YAML 프론트매터를 포함하여 메타데이터를 관리:

```markdown
---
code: "3-1-1"
title: "직제규정"
category: "제3편 학사행정 > 제1장 일반행정"
enacted_date: "YYYY-MM-DD"
last_revised: "2025-02-06"
status: "시행중"  # 시행중, 폐지, 삭제
related_regulations:
  - "1-0-1"  # 학교법인 단호학원 정관
  - "2-0-1"  # 용인대학교 학칙
revision_history:
  - date: "2025-02-06"
    description: "개정"
  - date: "2024-XX-XX"
    description: "이전 개정"
---

# 직제규정

제 1 장 총칙

제1조[목적] ...
```

## 3. 주요 기능

### 3.1 규정 검색 스크립트
```bash
# 키워드로 검색
python scripts/search.py "교원인사"

# 규정 코드로 검색
python scripts/search.py --code 3-1-1

# 최근 개정된 규정 확인
python scripts/search.py --recent 30  # 최근 30일
```

### 3.2 규정 업데이트 워크플로우
```bash
# 1. 해당 규정 파일 수정
vim regulations/3-학사행정/1-일반행정/3-1-1.md

# 2. 프론트매터의 last_revised와 revision_history 업데이트

# 3. 검증 실행
python scripts/validate.py regulations/3-학사행정/1-일반행정/3-1-1.md

# 4. 인덱스 자동 업데이트
python scripts/generate_index.py

# 5. Git 커밋
git add regulations/3-학사행정/1-일반행정/3-1-1.md docs/INDEX.md
git commit -m "개정: 직제규정 (3-1-1) - 2025-02-06"
git push
```

### 3.3 전체 규정집 생성
```bash
# 모든 규정을 하나의 파일로 병합 (필요시)
python scripts/merge_all.py --output complete_regulations.md
```

## 6. 즉시 시작 가능한 최소 구현
